十五、数据持久化

一、Pickle技术介绍：

pickle模块将内存中的Python对象转化成序列化的字节流，这是一种可以写入任何类似文件对象的字节串。Pickle模块也可以根据序列化的字节流重新构建原来内存中的对象：转换成原来那个对象。可以说pockle模块取代了专有的数据格式。他的序列化格式具有普遍性，对所有程序都足够高效。使用pickle，就不需要在持久化存储对象的时候手动地把对象转换为数据，也不需要在数据转化回对象时手动地解析复杂的数据格式。Pickle感觉上比较像XML的表示方法，但是，他是Python专用的方式并且比较容易编写代码。

2、更新Pickle文件中的对象

使用pickle更新数据有一个潜在的缺点，当处理非常大的数据库时会变得很慢。因为跟新一条记录需要读出和重新写入这个数据库，这比较浪费时间。

二、使用shelves

shelves自动将pickle进和pickle出，一个基于键访问的文件系统。他们很像必须打开者的字典，在程序员退出时进行持久化。因为它使用键访问被存储记录，所以不需要手动地为每一条记录管理一个普通文件，shelve系统自动分隔存储记录，并且只获取和更新被访问和修改的记录。通过这种方式，shelves使用起来就像一堆只存储一条记录的pickle文件，但是更容易编写代码。

四、XML可扩展标记语言简介：

XML是一种可按应用目标进行扩展的通用标记语言（可自定义标签）。旨在通过使用XML，使互联网数据共享变得更容易。

XML和HTML都是从标准通用标记语言SGML简化而成。与HTML相比，它对数据的记录方式做了特殊处理。

XML是用来存数据的，共享数据的，通用性比较好。

HTML关注呈现给用户的页面，XML关注内容包含的内容。

当前网络设备的导出配置大都是XML的格式。

五、JSON

1、JSON简介：

支持的类型：

string是0个或者多个字符的序列，用双引号括起来，支持转移

number支持证书负数正数小数

object：一个name/value pair，以{开始，以}结尾，name和value之间以：分割，每一个name/value pair以逗号分割。（与python字典类似）

array：是有序的value的集合，以[开始，以]结束，value之间用逗号分割。（类似列表）
