HTTP协议：

  • 超文本传输协议
  • 即超文本传输协议，是一种详细规定了浏览器和万维网服务器之间互相通信，万维网交换信息的基础，它允许HTML文档重web服务器传送到web浏览器。

http响应码：

200：客户端请求成功，是最常见的状态。

302：重定向

404：请求资源不存在，是最常见的状态。

401：请求未经授权

403：服务器收到请求，但是拒绝提供服务。

500：服务器内部错误，是最常见的状态。

503：服务器当前不能处理客户端的请求。

http url：

统一资源定位符，访问一个网站的ip地址。

可以通过Linux大小写敏感来判断对方的服务器的系统。

http头讲解：

200响应码

data：日期

content-type：类型

Accept-Charset：浏览器可接受的字符集

Content-Length：标识请求消息正文的长度

Cookie：这是最重要的请求头信息之一

get与post：

get是通过url来传递参数，post则不会通过url而是通过包的内容进行传递。

http请求方法：

GET 、POST、put、option、move、delete、trace

错误配置HTTP方法导致的安全事件：

服务器中存在允许put和move方式，这时我们可以通过put方式传一个webshell.txt然后通过move方式结合解析漏洞这样就可以很快的拿到网站的webshell。

在PHP中通常会使用$_SERVER[“HTTP_CLIETN_IP”]或者$_SERVER[“HTTP_X_FORWARDED_FOR”]来获取IP地址。

因此，我们就可以通过修改http头中的X-forwarded来进行攻击。
