2.面向对象：（大型程序，战略层面）

类是制造对象的工厂，制造出来的每个对象都是独一无二的，他们有着自己的属性。

类主要用于制造一个东西（程序）

二、类的特点：

  • structure：（结构）

通过OOP，我们可以把记录数据和其处理逻辑自然地结合起来，类既提供了程序单元将逻辑与数据合并在单独的包中，又提供了一个层级结构，使得代码更容易组织从而防止代码冗余。

  • Encapsulation（封装）

通过OOP，我们可以把名字处理和增加薪水等操作的细节封装在方法里。例如我们可以自由地更改方法的实现，而不会破坏调用者的代码。

  • Customization（定制）

通过OOP，我们拥有一条自然地成长之路。通过编写新的子类可以扩展和定制类，同时不会更改或者破坏已工作的代码。
