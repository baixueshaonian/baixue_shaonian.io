Linux管理操作系统

一、管理操作系统：

RHEL5和RHEL6：通过init进程来管理系统

RHEL7: 通过systemd进程来管理系统

二、Linux系统启动过程：–》init进程

1）BIOS基本输入输出系统：

​ A Post开机自检

​ B 根据用户设置的启动顺序进程启动（光盘、硬盘、USB等）

2）MIBR硬件跳转指针512字节

​ 第一部分：基本驱动和服务信息 446字节

​ 第二部分：磁盘分区表 64字节

​ 第三部分：MBR分区结束标识：AA55 标志2字节

3）引导菜单grub、lilo的加载

​ 加载内容：根分区在硬盘中的位置、内核的参数、内核镜像的位置。

grub的修改： 位于/etc/grub.conf文件中。

4）init进程的加载：

init集成是PID=1的父进程，所有的进程都是他的子进程。

作用：加载内核、初始化分区、激活网路&设备、开机自动启动服务、开机用户初始化操作、登录提示（init进程的子进程）

三、Linux系统启动级别：

0：关机（init 0 关机命令）

1：单用户模式（可对系统进行修复，修改root密码）

2：不带NFS的多用户模式（基本不用）

3：命令行

4：未使用

5：图形化

6：重启

可以使用runlevel来查看当前的级别

可以在/etc/ inittab文件中查看到当前的level

id：5：initdefault

如果想要修改对应的级别那么只要修改该数值即可

只要重启就可以生效。

四、linux系统启动详解

BIOS 自检：检查外设，寻找启动磁盘。

启动加载器 grub：grub共两个部分。第一部分存放在硬盘的1扇区，即主引导扇区（MBR）。MBR=446+64+2字节。主要工作是加载操作系统信息。第2个阶段调用/boot/grub.conf来启动内核和内存初始化盘。

kernel：内核启动后，首先会根据/boot/grub/grub.conf的设置以只读方式挂载/文件系统，然后执行/文件系统上的init进程。

init服务：当init进程启动后，kernel便自动隐退到内存中。init进程会根据/etc/inittab文件的设置依次执行rc.init、rc和rc.local三个文件，如果修改/etc/inittab文件设置，可执行：init q重新加载配置信息。

小结：

  1. kernel内核启动（挂载文件系统只读RO）

  2. 启动init进程

  3. 根据/etc/inittab的系统启动级别加载系统

  4. 依次执行rc.sysinit：初始化linux的环境、挂载文件系统（RW）、加载设备驱动、主机名等等。

  5. 依次执行rc脚本：会依据启动级别runlevel（/etc/inittab）设置建立和启动相应的环境。/etc/rc.d/rc*.d/ *是当前系统启动级别

  6. 依次执行rc.local

    用户配置的系统初始化脚本（如果为空则跳过）

    7.验证用户登录界面

五、Systemd简介（取代init进程）

systemd是Linux系统用于管理系统和服务的工具，他被设计为向后兼容SysV init脚本，并提供了大量的功能，包括在引导系统时并行启动系统服务，按需激活守护进程，支持系统状态快照，基于依赖关系的服务控制逻辑。

systemd是一个程序（可以并行执行，这是与init最大的不同）
